避免创建不必要的对象
一般来说，最好能重用对象而不是在每次需要的时候就创建一个相同功能的新对象。
如下反面例子：
String s = new String("DON'T DO THIS");
该语句每次被执行的时候都创建一个新的String实例，但是这些创建对象的动作全都是不必要的。
改进后：
String s = "DON'T DO THIS";
该语句只用一个String实例

除了重用不可变的对象之外，也可以重用那些已知不会被修改的可变对象。
反面例子见slowversion包
修改后见fastversion包

不要错误的认为“创建对象的代价非常昂贵，应该要尽可能地避免创建对象”，相反，由于小对象的构造器只做很少量的
显式工作，所以，小对象的创建和回收动作是非常廉价的，特别是在现代的JVM实现上更是如此。通过创建附加的对象，
提升程序的清晰性、简洁性和功能性，通常是件好事。
